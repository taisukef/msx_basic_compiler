MSX-BACON による MSX-BASIC の命令サポート一覧
===============================================================================

-------------------------------------------------------------------------------
【書式】
	ABS( <数値> )
【備考】
	<数値> の絶対値を返す。
	MSX-BASIC では、値の値域によって適切な型の値が返されますが、
	MSX-BACON では、演算結果は、引数として与えられた <数値> と同じ型を採ります。
	<数値> が整数型で、-32768 だった場合は、期待する値は得られない。
	-32768 を 32768 にしたい場合は、CSNG を使って型キャストを行っていただければ
	単精度実数型の 32768 が返されます。
【例】
	ABS( X )
	ABS( CSNG( -32768 ) )

	ASC( <文字列> )
【備考】
	文字列の一番最初の文字のASCIIコードを返す。

【例】
	ASC("A")

-------------------------------------------------------------------------------
【書式】
	BINDATA <2進数データ> [, <2進数データ> ...]
【備考】
	DATA文と同じようにデータを配置します。
	データは 0 と 1 の記号だけ使って記入して下さい。
	8文字単位で、1byte の数値に変換して、それを文字コードとする文字列に変換して
	データ化します。
	文字数が8の倍数でなければエラーになります。
	0,1以外の記号を使うとエラーになります。
	たとえば、下記のデータは、

	BINDATA 00110001001100100011001100110100

	下記と等価です。

	DATA "1234"

	コメントの記述は注意が必要です。

	BINDATA 01010101 ' comment

	この記述は、コンパイル時に Illiegal function call のエラーになります。
	BINDATA は、"01010101 ' comment" を、2進数データとして処理しようとして、
	2進数として認識できないスペースやコメント記号などを解釈できないためです。
	コメントを記入したい場合、: や "" で区切って下さい。

	BINDATA "01010101" ' comment
	BINDATA 01010101:' comment

	※BACON固有関数
【例】
	BINDATA 00111100
	BINDATA 01111110
	BINDATA 11111111
	BINDATA 11111111
	BINDATA 11111111
	BINDATA 11111111
	BINDATA 01111110
	BINDATA 00111100

-------------------------------------------------------------------------------
【書式】
	BLOAD <文字列式> [,R][,オフセットアドレス]
【備考】
	<文字列式>で書かれたファイル名のバイナリファイルをメモリに読み込みます。
	もし、HIMEM より若いアドレスにロードされるファイルであれば、
	BACONのコンパイル結果が破壊されていると見なし、ロード後に停止します。
	ただし、R がついている場合は、実行して戻ってこない想定で動作します。
	HIMEM より大きいアドレス(FFFFhに近い方)の場合、ロード後も戻ってくる想定
	で動作します。こちらも、R がついている場合は、実行しますが、戻る想定で
	動作します。
	HIMEM の値は、CLEAR の第２パラメータで指定できます。

	BACON のコンパイル結果ではなく、自作の機械語コードを &HC000 へ読み込む場合、
	そのようなヘッダがついている BINファイルを用意しておいて、
	BACON の方は、冒頭に CLEAR 100,&HC000 と書いておけば、MSX-BASIC の時と
	同様の動作になります。

	オフセットアドレスを指定すると、ファイルに記録されているアドレスに、その値
	を加算したアドレスへロードする。実行開始アドレスも同様にオフセットが加算
	される。
	オフセットアドレスには数式を指定できるが、変数R は使用できない。R と記入すると、
	自動実行の R と認識されるためである。

【例】
	BLOAD "TEST.BIN",R
	BLOAD "TEST.BIN",100
	BLOAD "TEST.BIN",R,100

-------------------------------------------------------------------------------
【書式】
	BLOAD <文字列式>, S [, オフセットアドレス]
【備考】
	BSAVEでファイルに保存された VRAMデータを、読み出す。
	<文字列式>でファイル名を指定する。
	オフセットアドレスを付けると、ファイルに記録されているアドレスに、その値を
	加算したアドレスへロードする。
【例】
	BLOAD "TEST.SC5",S
	BLOAD "TEST.SC5",S,1024

-------------------------------------------------------------------------------
【書式】
	BSAVE <文字列式>, <開始アドレス>, <終了アドレス> [, <実行開始アドレス>]
【備考】
	<開始アドレス> から <終了アドレス> までの範囲の RAM の内容を <文字列式> の
	ファイル名でファイルに書き出す。
	<開始アドレス> 及び <終了アドレス> は、書き出し対象に含まれる。
	<実行開始アドレス> は、BLOAD,R でロード後即実行する場合の実行開始アドレス
	である。省略すると、<開始アドレス> と同じ値が指定されたものとされる。

【例】
	BSAVE "HOGE.BIN", &H8000, &H8FFF

-------------------------------------------------------------------------------
【書式】
	BSAVE <文字列式>, <開始アドレス>, <終了アドレス> , S
【備考】
	<開始アドレス> から <終了アドレス> までの範囲の VRAM の内容を <文字列式> の
	ファイル名でファイルに書き出す。
	<開始アドレス> 及び <終了アドレス> は、書き出し対象に含まれる。

【例】
	BSAVE "HOGE.SC5", &H0000, &H7FFF, S

	を指定すること。

-------------------------------------------------------------------------------
【書式】
	CIRCLE [STEP] ( X座標, Y座標 ), 半径, 色, 開始角, 終了角, 比率
【備考】
	円を描画する。
	角度は、右が 0°、反時計回りに角度が進行する。
	開始角・終了角を負数にすると中心と線で結ぶ。値として負数の意味は無く、符号が
	そのまま中心と線で結ぶかどうかのフラグであることに注意すること。負にしたから
	と言って、時計回りになるわけでは無い。
	角度はラジアン単位で指定する。30° を指定したい場合は、30 * π/180 を指定する
	必要があるので注意。
	比率は楕円にする場合の縦横比である。傾いた楕円は描画できない。
	高速化のために、やや演算を省略している。MSX-BASICで描画した場合と比べ、
	若干歪な形状になる場合があるので注意すること。特に他のグラフィック描画命令と
	組み合わせて閉領域を作り、その中を PAINT で塗りつぶすようなケースでは、
	MSX-BASICと完全に同じ動作になるかは不明である。実際に動かしてみて確認する
	必要がある。
【例】

-------------------------------------------------------------------------------
【書式】
	CLEAR <文字列領域のサイズ>, <機械語領域のアドレス>
【備考】
	BACON では <文字列領域のサイズ> は無視されます。
	<機械語領域のアドレス> は HIMEM に書き込まれ、スタックと変数も初期化されます。
	<機械語領域のアドレス>以降のアドレスを機械語領域として利用可能になります。
	※以降と言っても、E000h以降は使わない方が良いです。DiskBIOS等のワークエリア
	として使われていて、機種によってそのアドレスが異なります。試しに使ってみて
	問題なく動いたからと行って、別の機種でも同様とは限りませんのでご注意下さい。
【例】

-------------------------------------------------------------------------------
【書式】
	COLOR [= {NEW|RESTORE| ( <PALETTE>, <RED>, <GREEN>, <BLUE> )}]
【備考】
	カラーパレットを変更する。MSX1で使用すると暴走するので注意。
	COLOR=NEW は、カラーパレットを初期化する。COLOR だけの実行でも同等である。
	COLOR=RESTORE は、VRAMのカラーパレットエリアに格納されているパレット情報に
	基づいてカラーパレットを変更する。
	COLOR=(P,R,G,B) は、カラーパレット P に対して、(R,G,B) の色を設定する。
	P は 0～15、R,G,B は 0～7 の範囲で指定。
【例】
	COLOR
	COLOR=NEW
	COLOR=RESTORE
	COLOR=(15,7,7,7)

-------------------------------------------------------------------------------
【書式】
	COLOR SPRITE( <スプライトプレーン番号> ) = <色>
【備考】
	スプライトプレーン番号の色を変更する。
	MSX-BASIC2.0 から追加された命令であるが、
	MSX-BACONでは、MSX1 でも動作するコードを生成する。
【例】
	COLOR SPRITE(0)=10

-------------------------------------------------------------------------------
【書式】
	COLOR SPRITE$( <スプライトプレーン番号> ) = <色>
【備考】
	スプライトプレーン番号の色を変更する。
	スプライトモード2専用の命令であり、SCREEN0～3では何もしない。
	（MSX-BASICでは、SCREEN0～3で実行するとエラーになる）
	<色>は文字列で与える。1文字1ラインに対応しており、先頭16文字が有効である。
	16文字に満たない分は、0 が与えられたものとして処理する。
【例】
	COLOR SPRITE$(0)=CHR$(10)+CHR$(5)+CHR$(5)+CHR$(5)+CHR$(5)+CHR$(5)

-------------------------------------------------------------------------------
【書式】
	COPY <ファイル名> [,<方向>] TO (X3,Y3) [,<ページ> [,<LOP>]]
【備考】
	ファイルの内容を、(X3,Y3) へ描画する。
	方向は、転送の向きで、MSX-BASIC で指定可能な 0～3 に加え、
	MSX-BACON では 4～5 が拡張になっている。
		0: 始点(X3,Y3) で右・下方向に描画
		1: 始点(X3,Y3) で左・下方向に描画
		2: 始点(X3,Y3) で右・上方向に描画
		3: 始点(X3,Y3) で左・上方向に描画
		4: 始点(X3,Y3) で右・下方向に描画 (※0 と同じ)
		5: 始点(X3+幅,Y3) で左・下方向に描画
		6: 始点(X3,Y3+高さ) で右・上方向に描画
		7: 始点(X3+幅,Y3+高さ) で左・上方向に描画

	4～7 を使うと、(X3,Y3) の左上とする位置に、画像だけ左右・上下に反転した
	描画を行えるため、向きを変えたパーツの描画にはこちらを使うと便利。
【例】
	COPY "TEST.GR5",5 TO (0,0)

【書式】
	COPY <ファイル名> TO <配列変数名>
【備考】
	ファイルの内容を、指定の配列変数に読み出す。
【例】
	DIM A(99)
	COPY "ARRAY.BIN" TO A

【書式】
	COPY <配列変数名> [,<方向>] TO (X3,Y3) [,<ページ> [,<LOP>]]
【備考】
	配列変数に格納されている矩形画像を、指定の位置へ描画する。
	方向は、転送の向きで、MSX-BASIC で指定可能な 0～3 に加え、
	MSX-BACON では 4～5 が拡張になっている。
		0: 始点(X3,Y3) で右・下方向に描画
		1: 始点(X3,Y3) で左・下方向に描画
		2: 始点(X3,Y3) で右・上方向に描画
		3: 始点(X3,Y3) で左・上方向に描画
		4: 始点(X3,Y3) で右・下方向に描画 (※0 と同じ)
		5: 始点(X3+幅,Y3) で左・下方向に描画
		6: 始点(X3,Y3+高さ) で右・上方向に描画
		7: 始点(X3+幅,Y3+高さ) で左・上方向に描画

	4～7 を使うと、(X3,Y3) の左上とする位置に、画像だけ左右・上下に反転した
	描画を行えるため、向きを変えたパーツの描画にはこちらを使うと便利。
【例】
	COPY CH, 5 TO (32,0)

-------------------------------------------------------------------------------
【書式】
	DATA <定数> [, <定数> ....]
【備考】
	データを配置する。
	注意すべきは、<定数> の部分は全て文字列として保持されるという点です。
	文字列データに : を埋め込みたい場合は、"" で囲って下さい。

	DATA "00:00:00"

	囲わない場合、: は DATA命令の終わりと判定されます。

	下記は、"data ' comment" という文字列のデータになります。

	DATA data ' comment

	つまり、' comment の部分も、文字列としてプログラムに埋め込まれます。
	' comment は、コメントとしたい場合は、明確に : で区切るか、"" で囲む必要があります。

	DATA "data" ' comment
	DATA data:' comment

	この記述であれば、' comment はコメントとして、コンパイル時に消滅します。

【例】

【備考】
	変数の型指定( %, !, #, $ )を省略した場合の型を指定する。
	省略表記のデフォルトは、倍精度実数型 # の扱いである。
	DEFINT は 整数型 %、DEFSNG は 単精度実数型 !、DEFDBL は倍精度実数型 #、
	DEFSTR は 文字列型 $ となる変数名の頭文字を指定する。

	DEFINT A

	これで、A から始まる名前の変数は、型指定をしなければ % を指定したことになる。
	A, A1, AB, AD 等。
	変数名指定は、"," で繋げて複数指定することができる。

	DEFINT A,B,C,D

	連続する場合、- で繋げて間の記述を省略できる。

	DEFINT A-D

	これらを組み合わせることもできる。

	DEFINT A-D,Q-T,Z

	, の前後では順不同だが、- の前後は同じか、右の方が Z に近くなければならない。
	従って、下記の記述はエラーである。

	DEFINT Z-A

	下記の記述は問題ない。

	DEFINT D,C,B,A

【例】
	DEFINT A
	DEFINT A,B,C,D
	DEFINT A-D
	DEFINT A-D,Q-T,Z
	DEFINT D,C,B,A

【書式】
	DIM <配列変数名> [<型識別子>] ( <要素1> [ ,<要素2> [...]] ) [, <配列変数名>... ]
【備考】
	配列変数のメモリを確保する。
	DIM A%(10) で、A%(0), A%(1), ... , A%(10) の 11要素の配列変数を使えるようになる。
	ただし、既に確保済みの配列変数に対して再度 DIM すると、Redimensioned array 
	の実行時エラーになる。サイズを変えたくなった場合は、一度 ERASE命令で削除してから
	再度 DIM しなければならない。削除した時点で内容は失われる。
	要素を , で区切ることにより、多次元配列も利用できる。
	ただし、同じ名前の配列変数は、１つのプログラム内で同じ次元数でなければならない。
	例えば、下記のようなケースはエラーである。

	DIM A%(1,2,3)
	ERASE A%
	DIM A%(1,2)

	最初は３次元で宣言し、次の宣言では２次元になっている。
	MSX-BASIC では許容される記述であるが、MSX-BACON ではコンパイル時にエラーになるので
	要注意。
	要素を囲う () は、[] でも代用できる。また、(] や [) などの組み合わせも可能であるが
	視認性が悪くなるため、() か [] を使うことを推奨する。
	型識別子を指定しなかった場合、DEFINT,DEGSNG, DEFDBL, DEFSTR の指定が有効であり、
	デフォルトでは倍精度実数である。

【例】
	DIM A(5), B(6), VX(8), VY(8)

【書式】
	FIELD #<ファイル番号>, <長さ> AS <文字列変数名> [, <長さ> AS <文字列変数名> ...]
【備考】
	ランダムアクセスモードのファイルに対して、対応する文字列変数とサイズを指定する。
	この命令は、MSX-BASIC とは若干挙動が異なるので注意。

	MSX-BASIC では、
		FIELD#1,10 AS A$,10 AS B$:FIELD#1,20 AS C$
	とすると、ファイル用のバッファ先頭 20byte のうち、最初の 10byte に A$、次の 10byte に B$
	最初の 20byte に C$ の名前を付けるため、同じメモリを複数の変数で共有することになる。
		LSET A$="1234567890":LSET B$="ABCDEFGHIJ":PRINT C$
	とすると、A$,B$の内容を書き換えたことで C$ の中身も書き換わり、
	"1234567890ABCDEFGHIJ" が表示される。
	複数回 FIELD を実行することで、いくらでも「重複」を指定できる。
	重複指定した変数全てに書き込んでいるのでは無く、アドレスを共通にしているだけなので
	重複の数は処理負荷には影響しないが、変数の実体のアドレスを書き替えてしまう場合がある =
	を使うと FIELD の設定は壊れてしまう。
	また、FIELD指定直後の文字列変数は、バッファの内容そのものを示しているため、不定である。
	MSX-BACON の構造上この重複の表現が困難であることと、そもそもこの仕様は極めて分かりづらい
	ため、挙動を変えている。

	MSX-BACON では、
		(1) FIELD を複数回実行すると後優先で上書きされ、以前の設定は消失する。
		(2) LSET, RSET だけでなく = による代入をしても FIELD の情報は壊れない。
		(3) FIELD で指定した変数は指定の文字数の " " で初期化される。
		(4) PUT, GET の際に文字列変数に個別にアクセスするため、若干遅い。
		(5) 指定できる変数は 16個までである。
		(6) 配列変数は指定できない。
	という点が、MSX-BASIC と異なる。

	サイズがレコードサイズを超えてはいけません。
	ベーコンの FIELD命令は、サイズチェックを行っていないため、レコードサイズを超えた場合
	挙動不審になりますのでご注意ください。
【例】
	FIELD #1,10 AS A$, 20 AS B$

【書式】

【備考】

	このとき、HEXCHR$() を通した結果をデータとして書き出します。
	たとえば、下記のデータは、

	HEXDATA 31323334,41424344

	下記と等価です。

	DATA "1234","ABCD"

	MSX-BASIC では、16進数を文字列で記入して、READ してから VAL("&H"+A$) のように
	変換する過程を経る使い方をよく見かけますが、その処理時間を節約することが出来ますし、

	コメントを記入したい場合、: や "" で区切って下さい。

	HEXDATA "31323334" ' comment
	HEXDATA 31323334:' comment

	※BACON固有関数
【例】
	HEXDATA 31323334,41424344

-------------------------------------------------------------------------------
【書式】 命令
	MID$( <文字列変数>, <切り出し開始位置>, <文字数> ) = <文字列2>
【備考】
	文字列変数の切り出し開始位置から指定の文字数だけを文字列2に置換する。
	A$ = "12345" の場合、MID$(A$,3,2)="ABC" を実行すると A$="12AB5" となる。
	-compatible , -original のオプションで動作が異なるので要注意。

	-compatible は、<文字列変数> に指定の変数を、<文字列2> に単独で指定すると
	上書き更新されて、先頭部分の繰り返しになる。

		A$="123456":MID$(A$,3)=A$:PRINT A$
		121212

	これが、3文字目以降に =A$ の A$ で置き換えるわけですが、123456 の 3 が 1
	に置き換わると、121456 になる。このようにして全文字置換すると、121212 になる。
	しかし、

		A$="123456":MID$(A$,3)=A$+"":PRINT A$
		121234

	=の右側が文字列式になっていると、式の結果が格納される別の領域が確保された上で
	そこに結果が格納され、A$ と A$+"" は違うメモリ領域を指していることになり、
	期待通りの結果が得られる。

	MSX-BASIC がもともとこのバグのような動作を持っているため、-compatible は、
	この挙動を忠実に再現する。

	一方で、-original では、上記２パターンでは、いずれも 121234 と期待通りの
	動作になる。

【例】
	MID$(A$,3,2)="ABC"

【書式】
	RETURN [<行番号>]
【備考】
	サブルーチン呼び出し元へ戻る。
	行番号を指定すると、戻りアドレスをスタックから除去して指定の行番号へ飛ぶ。

	複数の割り込みを同時に利用していて、かつ割り込みからの復帰に RETURN 行番号
	（行番号指定の強制リターン）を使うと、割り込み1を処理中に割り込み2が発生して
	割り込み2で RETURN 行番号 へ飛ぶと、処理中の割り込み1 へ戻らなくなり、
	再度割り込み1が入る。このようなケースでは割り込み1のスタック消費が解放されず
	積み上がっていき、スタックオーバーフローになります。
	通常のサブルーチンであっても、GOSUB/RETURNの対応関係が崩れた状態に
	ならないようご注意下さい。そのあたり、理解困難な場合は、RETURN 行番号 は、
	使うべきではありません。スタックオーバーフローすると、プログラムが暴走して
	不可解な動作になりますのでご注意下さい。
【例】
	RETURN
	RETURN 100

-------------------------------------------------------------------------------
【書式】
	VDP( <レジスタ番号> ) = 式
【備考】
	<レジスタ番号> で示されるVDPレジスタに対して、式の評価結果を書き込む。
	<レジスタ番号> に指定できるのは、下記の値。
		0～7 ..... VDP R#0～#7
		9～27 .... VDP R#8～R#26
		33～48 ... VDP R#32～R#47
	これ以外の値を指定した場合、何もしない。MSX-BASIC のように 
	Illegal function call にはならないので要注意。
	<レジスタ番号> には式を記述できる。VDP( 24+1 ) のような記述も可能である。
	また、式の値は下位 8bit のみ有効である。上位 8bit は無視される点も、
	MSX-BASIC とは異なる。（MSX-BASIC では上位8bit が 0 以外だと Illegal function 
	call になる。）

【例】
	VDP(24)=100

-------------------------------------------------------------------------------
【書式】
	VDP( <レジスタ番号> )
【備考】
	<レジスタ番号> で示されるVDPレジスタを読み出して、値を返す。
	<レジスタ番号> に指定できるのは、下記の値。
		-9～-1 ... VDP S#9～S#1
		0～7 ..... VDP R#0～#7
		8 ........ VDP S#0
		9～27 .... VDP R#8～R#26
	VDPのコントロールレジスタ R# は、書き込み専用であるため、この関数が返す値は
	最後に書き込んだ値のバックアップ値である。
	VDPのコマンドレジスタ R#32～R#47 は、書き込み専用であり、かつVDPコマンド実行
	とともに変化してしまうため、本関数ではサポートしていない。
	-9～27 の値を指定した場合、不定が返る。MSX-BASIC のように Illegal function call
	にはならないので要注意。

【例】
	A=VDP(24)
